03 / 10
The Secret Sauce

The Topology Manifest

Before generating anything, the agent extracts a semantic blueprint of the diagram — a structured manifest that becomes the ground truth for both generation and quality review.

Extracted Manifest

nodes[] Every node with its type — terminal, decision, process, subprocess
edges[] What connects to what — the full adjacency map
hero_candidate Highest-connectivity node — the natural focal point of the diagram
critical_path Entry → exit path — the story the diagram tells
subgroups[] Named clusters — logical groupings that should stay visually together

This manifest is the difference between "looks like a diagram" and "actually IS your diagram, just beautiful."

Dual Output

Two Variants. One Run.

Every diagram automatically produces two outputs — each optimized for a different use case.

🎯 Polished

  • Publication-quality, layout-faithful
  • Reference image passed with a "completeness guard" instruction
  • All labels must match. All connections present. Every node accounted for
  • Designed for accuracy and trust
"Drop this into your design doc and no one questions the content."

🎬 Cinematic

  • Full creative reimagining
  • Topology manifest is the ONLY input — no reference image
  • Hero element becomes dominant, central, striking
  • Layout is fully reinterpreted
"This is the thing you put on a slide and people actually look at."

Show both side-by-side. Let the user pick. Or keep both.

05 / 10
Quality Built In

8-Dimension Quality Review

The Cinematic output isn't just vibes. Every generation goes through a structured quality review — and the last three dimensions exist specifically for diagrams.

1 Content accuracy
2 Layout quality
3 Visual clarity
4 Prompt fidelity
5 Aesthetic consistency
6 Label fidelity — every text label matches the manifest
7 Structural accuracy — node count and connections verified
8 Color-category fidelity — semantic legends preserved

The last three are diagram-specific and exist precisely because beautiful ≠ correct. You shouldn't have to choose.

06 / 10
Image Reference Pipeline

How PNG Input Works

People naturally have existing images they want to do something with. Two cases come up a lot:

"I have a PNG of a diagram. Can you beautify it?"

Yes. The agent analyzes the PNG, extracts its topology, and runs the full Polished + Cinematic flow — just like it would for a .dot file.

"I want this new infographic to feel like this image I already have."

Also yes. Pass the image as a style reference. The generator sees the visual and takes the style cues, but generates new content.

The big question was: how do you tell these apart?

07 / 10
The Smart Part

Analyze-First Routing

Before

Keyword-based routing. If the filename has "diagram" in it, go to the beautifier. Fragile. Wrong.

Now

Analyze the PNG before routing. One nano-banana call answers three questions at once:

// Single VLM call does double duty analyze(image) → { is_diagram: true | false, content: "semantic summary", aesthetic: "colors, mood, typography" }

The result does two jobs simultaneously:

1. Routes the image

Diagram → beautifier agent
Not a diagram → infographic builder

2. Becomes Step 1 ground truth

No duplicate analysis needed — the routing call IS the analysis

The routing decision and the analysis are the same call. That's the elegance.

Diorama Mode

Small but delightful. The 3D aesthetics (Claymation, Lego) now support an opt-in diorama mode.

Instead of materializing information as objects (a checklist → a clipboard, a list → a wall), characters act out a process in a scene.
